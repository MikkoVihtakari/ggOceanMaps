Vignette example figures that require downloads/fetches (so they can't run on CRAN) are
**pre-rendered and hosted remotely**, then embedded via raw.githubusercontent URLs like
`https://raw.githubusercontent.com/MikkoVihtakari/ggOceanMapsLargeData/master/docs/<name>.png`.
This keeps the ggOceanMaps package CRAN-small. Convention established by `premade-maps.Rmd`;
Mikko confirmed (2026-06-16) he wants remote URLs, not bundling figures in the package.

Key paths on this machine:
- Local ggOceanMapsLargeData clone (where figures go): `<your ggOceanMapsLargeData clone>` — figures in `docs/`.
- Cached high-res data (datapath): `~/Documents/ggOceanMapsLargeData`.
- User raster (userpath): `~/path/to/GEBCO_2025.nc`.

**Workflow:** generate PNGs into the LargeData `docs/` folder, then commit & push that
repo separately — its figures must be live for the embedded URLs to resolve.
The bathymetry figures were generated by `dev/make_bathymetry_vignette_figs.R`.

Caveat: the local LargeData clone had lost its `.git`; re-link with
`git init` + `git remote add origin <url>` + `git fetch` + `git reset --mixed origin/master`,
then commit only the new files (working tree differs from remote in unrelated files like
`.gitignore`, `data/.gitattributes`, `data/dd_rbathy_cont.rda` — don't stage those).
